# 2026-04-20

## Unblock mac-app CI (macos-15 + stale test suite)

**Branch:** `taylorhou/fix-mac-app-ci` (merged as PR #23)

### Context
Earlier the same day I bumped `actions/checkout@v4 → v5` across the repo's
workflows (PR #22) to clear the Node 20 deprecation warning. For
`mac-app.yml` I deferred the bump because touching the file triggers its
own path filter (`.github/workflows/mac-app.yml`), and the workflow
had never actually run to completion against the current source tree —
so merging the bump would surface a pile of pre-existing breakage. I
wanted to handle that separately, and this session is that cleanup.

### Three independent pre-existing breakages

**1. Runner.** `macos-14` ships Swift 5.10. `mlx-swift-lm` now pins
Swift ≥ 6.1. First compile step failed with:
```
package 'mlx-swift-lm' @ 8c9dd639… is using Swift tools version 6.1.0
but the installed version is 5.10.0
```
Fix: `runs-on: macos-15` (Xcode 16 / Swift 6.x).

**2. CreditKit refactor drift.** The source refactor that renamed
`Credit*` → `USDC*` had never propagated into the test suite. `CreditAmount+USD.swift`
is a gravestone that now reads simply *"Removed — USDCAmount IS USD.
No conversion layer needed."*

Rename table (all source-side, all months ago):
| old | new |
|---|---|
| `CreditAmount` | `USDCAmount` |
| `CreditPricing` | `InferencePricing` |
| `CreditLedger` | `USDCLedger` |
| `CreditTransaction` | `USDCTransaction` |
| `CreditAnalytics` | `USDCAnalytics` |
| `CreditSummary` | `USDCSummary` |

`WalletKitTests.swift`: trimmed to just the still-meaningful Base58 +
SolanaIdentity tests. The whole `CreditAmount USD Conversion Tests`
section went — "USDCAmount IS USD" means there's no conversion to test.
Dropped `@testable import CreditKit` since it was only needed for those
deleted assertions.

`CreditKitTests.swift`: rewritten against the current API. Dollar
amounts had to shift with the peg:

- welcome bonus: `100.0 credits` → `$0.01` (the literal value in
  `InferencePricing.welcomeBonus`)
- earn ratio: `0.80` (old 80/20 split) → `1 − platformFeeRate = 0.982`
  (1.8% network fee — matches `WalletKitConfig.platformFeeRate`)
- cost formula: now divides by `10_000`, so `1000 tokens × 1B × q4`
  resolves to `0.00001` USD instead of `0.1` credits
- ledger balance assertions adjusted proportionally (e.g. bonus + 10.0
  credit = `10.01`, was `110.0`)

**3. AgentKitTests.** `DelegationRule.maxCreditSpend` was renamed to
`maxSpend` (5 call sites; bulk-renamed via `sed`). Two
`XCTAssertEqual` sites tripped Swift 6's type inference:
```
error: type 'Equatable' has no member 'formal'
error: type 'Equatable' has no member 'autoAccept'
```
Swift 5 inferred the enum from the LHS; Swift 6 no longer does inside
`XCTAssertEqual`'s `<T: Equatable>` param. Fix: qualify.
```swift
XCTAssertEqual(decoded.preferences.tone, CommunicationTone.formal)
XCTAssertEqual(decision, NegotiationDecision.autoAccept)
```
