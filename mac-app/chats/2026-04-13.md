## Session 2: Multi-device Deployment, WAN Fixes, Hermes Integration

### What was done

**Deployed Teale to 3 devices:**

- Mac Mini 16GB (`ochou`) — Consumer node, LAN cluster + WAN enabled, Hermes agent connected

**App icon:**

- Generated .icns and wired into bundle.sh

**Solana wallet UI:**
- Added Settings section: enable toggle, devnet/mainnet picker with wallet address and balance display
- Network switch reinitializes the wallet bridge

**WAN peer models in Chat + API:**
- Chat model picker shows "WAN Peers" section with models from connected peers
- `/v1/models` API includes WAN and cluster peer models with `owned_by: "wan:<peer>"` or `"lan:<peer>"`
- `PeerModelProvider` closure in HTTP server aggregates peer models

**OTP fix:**
- Fixed force-unwrap crash in `AuthManager.verifyPhoneOTP` — safely unwraps session response

**WAN reliability fixes:**
- Suppressed non-fatal STUN errors from error display (relay fallback works fine)
- Increased reconnect attempts from 5→20 with shorter initial delay (3s) and gentler backoff (1.5x, max 30s)
- Detect stream completion from `finish_reason` in chunk (handles lost `InferenceComplete` messages)
- Persisted `allowNetworkAccess` via UserDefaults

**Inference routing:**
- Added LAN cluster priority: try ClusterProvider before WAN when no local model loaded
- Mac Mini Hermes → Teale localhost → LAN cluster → Mac Studio Qwen3-32B (confirmed working)

**Hermes configuration:**
- Mac Mini Hermes configured to hit Mac Studio directly over Tailscale (`100.88.240.125:11435`) for reliability
- Added `/no_think` to system prompt — Qwen3 reasoning chain was causing 14s responses, now ~1.6s
- Changed personality from `kawaii` to `helpful`

**Max output tokens:**
- Increased default max output tokens from 2048 → 8192
- Qwen3-32B natively supports 40,960 token context window — fits comfortably in 64GB with ~30GB headroom for KV cache

### Key Decisions
- **Direct HTTP over Tailscale** preferred over WAN relay or LAN cluster routing for Hermes. The WAN relay connections drop intermittently and LAN cluster connections are unreliable when restored from persisted state. Direct HTTP to the Mac Studio is rock solid.
- **`/no_think` for Qwen3** — The model generates hundreds of reasoning tokens for trivial questions. Disabling thinking mode gives 9x speedup with no quality loss for most tasks.
- **`scp` doesn't replace running binaries** — Discovered that `scp` silently fails to overwrite an .app while it's running. Must `killall -9` and `rm -rf` before deploying.

### Issues / Known Limitations
- **WAN relay connections are unreliable** — Peer connections drop and reconnect attempts often exhaust. The relay WebSocket stays connected but data connections between peers are fragile, especially across NAT (symmetric NAT on Mac Studio).
- **LAN cluster connections drop** — Bonjour discovery works but TCP peer connections don't always re-establish after the initial connection. Need to investigate the ClusterManager connection lifecycle.
- **Smart model routing not built yet** — Hermes can't route simple requests to a local small model and complex ones to the Mac Studio. Config option exists in Hermes (`smart_model_routing`) but needs both models on the same endpoint. Could build this into Teale's API layer.
